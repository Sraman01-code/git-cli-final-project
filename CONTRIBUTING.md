# Contributing to git-cli-final-project

All contributions, bug reports, bug fixes, documentation improvements,
enhancements, and ideas are welcome!

Thank you for considering contributing to this project. This document explains
the process for contributing.

## How to Contribute

1. **Fork the repository.** Click the "Fork" button on the repository page (or
   run `gh repo fork Sraman01-code/git-cli-final-project --clone`) to create
   your own copy of the project.

2. **Clone your fork** to your local machine:

   ```bash
   git clone https://github.com/<your-username>/git-cli-final-project.git
   cd git-cli-final-project
   ```

3. **Create a branch** for your change, with a short descriptive name:

   ```bash
   git checkout -b my-feature-or-fix
   ```

4. **Make your changes.** Keep changes focused — one bug fix or feature per
   branch. Update documentation if your change affects it.

5. **Commit your changes** with a clear commit message:

   ```bash
   git add .
   git commit -m "Short description of the change"
   ```

6. **Push the branch** to your fork:

   ```bash
   git push origin my-feature-or-fix
   ```

7. **Open a Pull Request** from your branch to this repository's `main` branch,
   describing what you changed and why.

## Reporting Bugs and Suggesting Ideas

- Open a GitHub **Issue** describing the bug or idea.
- For bugs, include the steps to reproduce, what you expected to happen, and
  what actually happened.
- For enhancements and ideas, describe the use case and the proposed behavior.
